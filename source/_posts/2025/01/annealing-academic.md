這是筆者上完這學期課後的唯一心得。想不到這學期的繁忙程度甚至跟上學期相當。相較於中央的朋友，筆者這學期比他們多了 ~3 科期末考[^1]和 2 個 final project，還要頂著演算法地獄、以及差點開天窗的 ML final project 壓力過完這學期。

總之，可喜可賀的是 GPA 沒有炸掉，筆者真是精算師 {% spoiler 誤 %}。

## 大重點

### 為什麼演算法是大魔王

&emsp;&emsp;筆者選的是**韓永楷**老師開的演算法。起初只是看到心得中說這位老師教的比隔壁班好很多，但卻忽略了文句中**給分不甜**這四字背後的沉重表現：在三段滿分分別為105/105/120的前提下，平均只有50.2/37.6/59.3，而且***不調分***，造就了以下奇景：

![](algo_dist.avif)

> 平均 GPA 只有 2.134；百分比平均64.28，標準差18.41

&emsp;&emsp;造就低分數的原因之一是因為演算法每題都相當在寫申論題：演算法過程、正確性證明、複雜度證明三項皆不可或缺，第一項寫不出來或寫錯基本上就是實實在在的0分。另外一個原因是批改過程相當嚴格，只要有一處寫得不嚴謹就等著被扣爛，**去要分甚至會被倒扣**[^2]。最大原因是不調分的給分方式：$\text{Max}(20+\text{avr}, \text{best two}\cdot 0.8 + \text{the other}\cdot 0.2)$，也就是純考試定勝負。細節可以看 Dcard 上有人寫的[更加詳細的文][algo_dcard_post]。

&emsp;&emsp;期中那鬼成績出來後筆者某朋友就果斷二退了，留下因為退了會低修所以沒有退路的筆者硬是跟另外一位電神朋友撐過了這學期。最終學期成績出來時聽說隔壁班高了10還20分。幸虧筆者在期中時就已經算好自己這科就算爛掉，GPA 也有 4.1 多的保底資訊了，所以壓力少掉不少(?)。最後也是如此。

&emsp;&emsp;但不得不說，老師教課邏輯與氛圍真的是相當好，再複雜的東西經過說明也能馬上理解吸收，可惜配上了這讓大多數人退卻的分數。{% spoiler 聽說是因為前陣子疫情期間被集體作弊搞過，`信任度--;` %}

### Final project 在做啥

&emsp;&emsp;這學期有兩個 final project，分別是 ML 與 OS 的。其中 OS 難度相較低很多，而且只要照訂好的進度走就好了，先不提。至於 ML 的部分，是在學期初就要訂一個自由研究的主題，筆者這組最後決定弄一個可以將渲染出來的 LaTeX 圖片反推回去語法的 NN。最後架構是採用 CRNN + Attention 下去做（不然根本 train 不起來 ##）。CNN 用 variable-sized input 最後接個 GAP；RNN 是用 LSTM。

&emsp;&emsp;上述聽起來很直接，但過程差點就開天窗。因為主題跟架構都是學期初就要決定，但那時我們還在學基本的 NN，花了成堆時間才搞懂 CRNN 是啥（這已經是學期末才會教到的主題了）。實作時又遇到一堆問題：tensorflow 不支援 Windows、看不懂 code、chatgpt 大法出來的也幾乎不會改，讓重大 bug 一直存活到中後期才被筆者挑出來（癱）。最後改了一下之前人用的 CNN 部分架構才跑出比較好的成果 owo。

> 聽說前幾屆抄襲也蠻嚴重的，repo 還是不要公開好了

#### 突然蹦出的 side-project

## 上課心得

### 隨興的密碼學

> CS 330500 | 密碼與網路安全概論
> Cryptography and Network Security

Cons:

&emsp;&emsp;要不是筆者期末忙到像甚麼一樣，想來寫期末評量時發現時限已經過的話，不然這科肯定會寫爛。先點破這堂課幾個有問題的地方：

1. 助教批改時不知變通：不論是在DE還是丟番圖方程，只要是**特解**都不唯一。期初考時助教卻死板板地照著教授給的答案改，不是上面寫的特解都不給分；**讓人懷疑是不是沒學過數學**，還是不懂得批判性思考全聽教授的話（雖然後來有改，至少會聽學生的觀點）。導致期初時大家要分要超久，場面極其混亂。期末考時有關**複雜度**的用詞根本就錯了，就算有學生寫甚麼 $O(2^{56})$ 也不應該算分，而不是納入正確答案中，否則唬爛個 $O(1) \equiv O(2^{56})$ 也沒問題。此外，只要沒明確寫出正確數字就依然不算分，即使把常數值全寫出來，死板到家。
2. 教授出課堂作業非常隨興：幾乎沒有預先訂好的作業，都是剛好講到甚麼東西就出成作業。我們這班剛好有位問問題相當有水準的人（像是證明課本上的通式確實包含丟番圖方程的所有解），大部分時間都是教授把她的問題直接拿來當作作業。如果沒有這位同學，我相當懷疑原本的作業主題應該會是甚麼。

&emsp;&emsp;有一次作業還出了要對其他人做社交工程的奇妙作業。作業主題是好的但細節還是沒有考慮到，最後因種種原因取消了。[^4]

> 詳情可以看[這篇文][crypto_dcard_post]。

Pros:

&emsp;&emsp;不過助教確實也有些還算好的地方，例如不在期末另外加作業，還有ppt做得還算用心。隨興走向的上課方式也讓上課涼度升高，上到一半會以為自己在修通識課。總體來說也確實有學到東西，雖然期初還有加簽考（沒發揮到甚麼作用），~~跟日本製的壓縮機一樣~~非常稀有。

### 還算有趣的 ML

> CS 460200 | 機器學習概論
> Introduction to Machine Learning

&emsp;&emsp;因為筆者在之前有嚴重質疑/瞎猜 ML 能力上限與運作原理的毛病，所以打算自己來修一次課一探究竟。總體來說這疑問還是沒有破除，但有減輕一點（有鑑於我們學的幾乎都是初級架構，還有看了幾個最近時事後得出的結論）。

&emsp;&emsp;教授還蠻接地氣的，也有顏值。因為普通作業是要用助教給的 template 手刻 NN，相當需要討論；再加上 final project 的部分，所以這課比較吃隊友能力。課修到中期後，每次作業要花個 1, 2 天寫好架構，**做好資料處理後**[^5]重複以下事項：

1. 隨便 train 好 basic part
2. 丟 kaggle
3. 看結果，爛掉的話回到 (1)
4. 改 advanced part 的 model
5. 開始 train，死魚眼地盯著 training loss 有沒有掉
6. 看 validation
7. 沒有比較好的話回到 (4)
8. 有潛力的話丟 kaggle
9. 看結果，爛掉的話回到 (4)，還有找 chatgpt 求救
10. commit checkpoint，想繼續卷 performance 的話回到 (4)
11. 累死

> 如果你是負責模型的部分，這輪迴會在弄 final project 的時候變成日常（丟 kaggle 以外的部分）。

### 終於修到的重訓

&emsp;&emsp;重訓在清大是知名極其難搶到的體育課之一，每學期必定會搶加簽搶破頭。筆者來清大兩學期後才搶到，大歡喜 owo。上課方式是每堂教 1~2 個身體的肌群和其拮抗肌、各部位要用甚麼對應的器材去訓練、和器材的正確使用方式以及身體姿勢。

&emsp;&emsp;期中考是 random 挑一個肌群，然後要自己找出其和其拮抗肌的器材然後進行操作。器材挑對就直接及格，再來才是看操作的姿勢。期末考是硬舉，會特別注重在身體姿勢上，再來才是看舉多重。老師人很好，沒看過他生氣；硬舉時也會在一旁看你動作有沒有做對。整體來說學到的東西相當多（對筆者這種室內派來說），推推。

{% note success %}
成績：A | 6/42
{% endnote %}

### 發現白做(X)的服務學習

> UMC After-school Tutoring

&emsp;&emsp;因為大二上筆者耍笨沒有及時停損讓服學被當，所以到手的書卷飛了；大二下發現自己服學又不會過時就馬上停損，書卷獎順利到手。這學期沒有意外的話書卷也會`++;`。雖然很難相信是從一個大三老人口中說出來，但筆者發現自己有點太注重課業了，導致生活重心完全其之上，每天都是教室宿舍兩點一線，極少專注在自身正常生活上。一部分是因為要走推甄；另一部分是 loading 真的還是偏重，不得不弄；另外一部分可能是筆者的自我要求過高，給自己太多壓力。所以下學期筆者打算調整一下 loading 比重，修一些養老的課讓自己可以轉移生活重心。
